## Documentation Standard

All documents must include:

- Version history
- Document owner
- Review status
- Approval status
- Scope
- Assumptions
- Risks
- Future scope

## Writing Rule

No filler content. Every page must help product, engineering, QA, DevOps, design, business or operations teams.
